* HARMer DRL: Simulation-Driven RL for Automated Penetration Testing

This repository contains a full research pipeline for reinforcement learning (RL)
in automated penetration testing. It includes:
- multi-subnet simulation environments,
- baseline and extended RL agents,
- reward-failure diagnosis and fixes,
- generalization and few-shot adaptation experiments,
- real-experiment preparation and live-lab execution scripts,
- paper artifacts (figures, tables, and IEEE-style manuscript source).

* Repository Highlights

** Core simulation and RL training
- `pentest_env.py`, `pentest_env_complex.py`, `pentest_env_fixed.py`
- `train_q_learning.py`, `train_dqn.py`, `train_a2c.py`, `train_complex.py`
- `compare_complex.py`, `evaluate_transfer_networks.py`, `generalize_and_adapt.py`

** Extended methods
- A3C-like: `a3c_pytorch.py`, `train_a3c_like_only.py`
- DPPO-like and GAIL-like: `simulation_extensions.py`, `train_simulation_extensions.py`

** Real-experiment assets
- Network configs and runbooks: `real_experiments/configs/`, `real_experiments/runbooks/`
- Live/dry-run orchestrator: `real_experiments/run_real_experiment.py`
- Result logs: `real_experiments/results/`

** Paper artifacts
- Manuscript source: `paper_overleaf/main.tex`
- Bibliography: `paper_overleaf/references.bib`
- Figures used in paper: `paper_overleaf/figures/`

* Real-Experiment Workflow

This project separates simulation claims from live execution claims.

** Dry-run validation
#+begin_example
cd real_experiments
python run_real_experiment.py --network alpha --mode dry-run
#+end_example

** Live lab run (authorized environment only)
#+begin_example
cd real_experiments
python run_real_experiment.py --network lab --mode live
#+end_example

Docker lab assets are under `docker_env/`.
